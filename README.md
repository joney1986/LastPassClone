# LastPass Clone - Mobile Password Manager

This project is a full-stack mobile application that clones the core functionality of a password manager like LastPass, built with a React Native frontend and a Node.js (Express) backend.

## ✨ Features

- **Secure User Authentication:** Standard login/registration flow.
- **Two-Factor Authentication (2FA):** Time-based One-Time Password (TOTP) for enhanced security.
- **Biometric Unlock:** Use Face ID or Touch ID to unlock the app.
- **Zero-Knowledge Architecture:** Employs a Master Password system to ensure all user data is encrypted and decrypted on the client-side. The server never has access to the plaintext vault key.
- **Password Management:** Full CRUD (Create, Read, Update, Delete) functionality for passwords.
- **Password History:** Keeps a log of previous passwords when an entry is updated.
- **Secure Notes:** A separate section for creating and managing encrypted text notes.
- **Client-Side Search:** Instantly search and filter passwords and notes on the device.
- **Modern UI/UX:**
    - A consistent design system for a clean, calm user interface.
    - Pull-to-refresh on lists.
    - Loading indicators for a responsive feel.

## 🛠️ Tech Stack

- **Frontend:**
  - React Native (with Expo)
  - React Navigation for routing and navigation.
  - `expo-crypto` & `crypto-js` for client-side cryptography.
  - `expo-local-authentication` for biometrics.

- **Backend:**
  - Node.js
  - Express.js
  - SQLite for the database.
  - `jsonwebtoken` for session management (JWT).
  - `bcrypt` for hashing login passwords.
  - `speakeasy` for 2FA token generation and verification.

## 🚀 Getting Started

Follow these instructions to get a copy of the project up and running on your local machine for development and testing purposes.

### Prerequisites

- [Node.js](https://nodejs.org/) (v16 or later recommended)
- `npm` or `yarn`
- [Expo Go](https://expo.dev/go) app on your mobile device for testing the frontend.

### Backend Setup

1.  **Navigate to the backend directory:**
    ```bash
    cd backend
    ```
2.  **Install dependencies:**
    ```bash
    npm install
    ```
3.  **Set up environment variables:**
    - Create a `.env` file in the `backend` directory by copying the example file:
      ```bash
      cp .env.example .env
      ```
    - Open the new `.env` file and replace the placeholder values with your own secure, random secrets.

    #### Database Setup
    The application uses SQLite, and the database is configured to be created automatically. When you start the server for the first time, a `db.sqlite` file will be created in the `backend` directory and all necessary tables will be set up. No manual database configuration is needed.

4.  **Start the backend server:**
    ```bash
    npm start
    ```
    The server will be running on `http://localhost:3000`.

### Frontend Setup

1.  **Navigate to the frontend directory:**
    ```bash
    cd frontend
    ```
2.  **Install dependencies:**
    ```bash
    npm install
    ```
3.  **Start the development server:**
    ```bash
    npx expo start
    ```
4.  **Run the app:**
    - Scan the QR code generated by the Expo CLI with the Expo Go app on your iOS or Android device.
